# Configuração do Tailwind CSS e Shadcn UI

Este tutorial descreve os passos para configurar o Tailwind CSS e integrar os componentes do Shadcn UI em um projeto React com Vite.

## 1. Configuração do Tailwind CSS

O Tailwind CSS é um framework CSS utilitário que permite construir designs personalizados diretamente no seu markup.

### 1.1. Instalação das Dependências

Primeiro, instale o Tailwind CSS, PostCSS e Autoprefixer como dependências de desenvolvimento:

```bash
npm install -D tailwindcss postcss autoprefixer
```

### 1.2. Inicialização do Tailwind CSS

Em seguida, gere os arquivos de configuração `tailwind.config.js` e `postcss.config.js`:

```bash
npx tailwindcss init -p
```

### 1.3. Configuração do `tailwind.config.js`

Edite o arquivo `tailwind.config.js` para especificar os arquivos que o Tailwind deve escanear em busca de classes CSS. Certifique-se de incluir todos os seus arquivos de componentes e páginas:

```javascript
// tailwind.config.js
/** @type {import('tailwindcss').Config} */
export default {
  content: [
    "./index.html",
    "./src/**/*.{js,ts,jsx,tsx}",
  ],
  theme: {
    extend: {},
  },
  plugins: [],
}
```

### 1.4. Inclusão das Diretivas Tailwind no CSS

Adicione as diretivas `@tailwind` no seu arquivo CSS principal (geralmente `src/index.css` ou `src/App.css`) para injetar os estilos base, componentes e utilitários do Tailwind:

```css
/* src/index.css */
@tailwind base;
@tailwind components;
@tailwind utilities;

/* Seus estilos personalizados podem vir aqui */
```

## 2. Configuração do Shadcn UI

O Shadcn UI é uma coleção de componentes reusáveis que podem ser copiados e colados diretamente no seu projeto. Ele utiliza Tailwind CSS para estilização e Radix UI para acessibilidade.

### 2.1. Inicialização do Shadcn UI

Para iniciar o processo de configuração do Shadcn UI, use o seguinte comando interativo:

```bash
npx shadcn-ui@latest init
```

Durante a execução, você será guiado por algumas perguntas. As escolhas comuns para um projeto React com TypeScript e Tailwind incluem:

- **Would you like to use TypeScript (recommended)?** `yes`
- **Which style would you like to use?** `Default` (ou outro de sua preferência)
- **Which color would you like to use as base color?** `Slate` (ou outro de sua preferência)
- **Where is your global CSS file?** `src/index.css`
- **Would you like to use CSS variables for colors?** `yes`
- **Where is your `tailwind.config.js` located?** `./tailwind.config.js`
- **Configure the import alias for components:** `@/components`
- **Configure the import alias for utils:** `@/lib/utils`
- **Are you using React Server Components?** `no` (para a maioria dos SPAs)
- **Write types to components.json?** `yes`

Este comando configurará o arquivo `components.json` e fará as integrações necessárias com o Tailwind CSS.

### 2.2. Adicionando Componentes Shadcn UI

Após a inicialização, você pode adicionar componentes individuais. Por exemplo, para adicionar os componentes `Button`, `Input`, `Label` e `Form` (que foram usados neste projeto), você usaria:

```bash
npx shadcn-ui@latest add button input label form
```

Isso irá adicionar os arquivos TypeScript/TSX desses componentes na pasta `src/components/ui/` (ou no caminho configurado para `@/components`).

### 2.3. Entendendo a Estrutura dos Componentes Shadcn

Os componentes Shadcn utilizam algumas bibliotecas e padrões importantes:

- **Radix UI:** Para funcionalidades e acessibilidade (ex: `LabelPrimitive` de `@radix-ui/react-label`).
- **`class-variance-authority` (cva):** Para gerenciar variantes de classes Tailwind de forma condicional e tipada (ex: `buttonVariants`).
- **`clsx` e `tailwind-merge`:** Para combinar classes Tailwind de forma inteligente, resolvendo conflitos.
- **`cn` utility:** Uma função auxiliar (geralmente em `src/lib/utils.ts`) que combina `clsx` e `tailwind-merge` para criar strings de classes dinâmicas.

Exemplo de uso (`Button` em `src/App.tsx`):

```typescript
// src/App.tsx
import { Button } from './components/ui/button';
// ...
<Button onClick={() => navigate("/company/register")}>Cadastrar Empresa</Button>
// ...
```

Exemplo de uso (`Form` e `Input` em `src/pages/Company/Register/index.tsx`):

```typescript
// src/pages/Company/Register/index.tsx
import { Input } from "@/components/ui/input";
import { Form, FormControl, FormField, FormItem, FormLabel, FormMessage } from "@/components/ui/form";
// ...
<Form {...form}>
  <form onSubmit={form.handleSubmit(onSubmit)}>
    <FormField
      control={form.control}
      name="name"
      render={({ field }) => (
        <FormItem>
          <FormLabel>Nome da Empresa</FormLabel>
          <FormControl>
            <Input placeholder="Nome da sua empresa" {...field} />
          </FormControl>
          <FormMessage />
        </FormItem>
      )}
    />
    {/* ... outros campos ... */}
    <Button type="submit" disabled={isLoading}>
      {isLoading ? 'Cadastrando...' : 'Cadastrar'}
    </Button>
  </form>
</Form>
```

Ao seguir esses passos, você terá um ambiente de desenvolvimento robusto com Tailwind CSS para estilização flexível e Shadcn UI para componentes acessíveis e facilmente personalizáveis.
